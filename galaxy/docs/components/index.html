Galaxy Grid

To create a 12 column row, simply initiate a section. Then, add a column number from 1 to 12 on the element's g attribute to specify the width of the column.

3
9
<section>    <div g="3">3<div>    <div g="9">9<div></section>

Automatic rows

Sometimes you'll have a lot of rows, and nesting them in multiple sections can be verbose. This is why Galaxy columns that add up to 12 will automatically start at the next row. The example below demonstrates columns overflowing two rows:

3
9
First Half
Second Half
<section>    <div g="3">3<div>    <div g="9">9<div>    <div g="6">First Half<div>    <div g="6">Second Half<div></section>

Nested Columns

Each row contains 12 columns by default, but you can make a more complex layout by nesting section within other sections.

6
6
9
<section>    <div g="3">3<div>    <div g="9">9<div></section>

Distributing spacing

Offsets

Offsets are used to apply a distinct margins that push everything else down. Like all columns, anything that overflows 12 will start in a new row.

9 offset-3
2 offset-10
6 offset-6
<section>    <div g="9 offset-3">size 9 with offset 3<div></section>

Position and Alignment

You can either apply positions to containers or directly on elements themselves. Galaxy components inherit the alignment of the container by default.

left
centered
right
<button g="left">Left Button<button><button g="centered">Centered Button<button><button g="right">Right Button<button>

Responsiveness

The example below holds four, sized 3 columns that will span 12 columns on screens with a width of the md breakpoint.

3 md-12
3 md-12
3 md-12
3 md-12
<section>    <div g="md-12">12 on md screens<div>    <div g="md-12">12 on md screens<div>    <div g="md-12">12 on md screens<div>    <div g="md-12">12 on md screens<div></section>

Galaxy Grid will flow columns in a new row automatically, but it's good practice to specify when exactly things will flow. To specify, just add breakpoint-width to the g tag.

Containers

It is recommended that you use containers instead of defining columns on a parent element. Not only is it more semantic, but it also adds consistency to your design.

<section>    <div g="xsm-container">xsm-container<div>    <div g="sm-container">sm-container<div>    <div g="md-container">md-container<div>    <div g="lg-container">lg-container<div>    <div g="xlg-container">xlg-container<div></section>

Forms and Text Inputs

Create a form like you normally would. Styles are applied automatically.

Inline Inputs

<form action="#">    <input type="email" placeholder="Your email" required>    <input type="password" placeholder="Your password" required>    <input type="submit" value="Sign in" required></form>

Grouped Inputs

<form action="#" g="column">    <input type="email" placeholder="Your email" required>    <input type="password" placeholder="Your password" required>    <input type="submit" value="Sign in" required></form>

Textareas

<textarea rows="10" placeholder="This is a textarea">

Tooltips

Creating tooltips is as simple as adding data-tooltip containing the content of the tooltip to any component.

Hover over me
<a data-tooltip="This is a Galaxy tooltip.">
